План работ

+1. Сделать грубый каркас-макет сайта со статичными заглушками вместо страничек. Должна работать навигация, должны
присутствовать все необходимые странички, чтобы можно было представить структуру сайта и уже начать с ней работать.
Готово.

2. Продумать структуру данных. Какие будут сущности, какие будут поля данных, какие связи и т.п.
Сущности:
users
    _id: uuid
        name: string
        hasReadWelcome: bool
        hasReadHelp: bool
        schedule: uuid
        currentWorkout: uuid
        finishedTraining: bool

schedules
    _id: uuid
        plannedWorkouts: [
            {date: timedate, workout: uuid},
            {date: timedate, workout: uuid},
            {date: timedate, workout: uuid},
            ...
        ]

workouts
    _id: uuid
        name: string
        motionType: string //горизонтальный жим, вертикальный жим, горизонгтальная тяга, ...
        activityType: string //силовая, вспомогательная, аэробная, ...
        bodyParts: array // [спина, трицепс, ...]
        level: number //уроень сложности: 1, 2, 3

Связи:
user -> schedule (1:1)
user -> workout (1:1)
schedule -> workout (1:*)


3. Набросать макетные данные в соответствии со структурой, чтобы дальше уже можно было с этими данными как-то работать.

4. Начать реализовывать функционал сайта

5. Сделать качественный контент. Написать все текста, заполнить все тренировки и т.п. Желательно как можно ближе к
финальной версии.

6. Сделать раскладки и макеты для дизайна. Сделать дизайн. Возможно заказать макет, пиктограммы и векторные рисунки.
Фото можно найти на фотобанках.

7. Финальная проверка, ловля и исправление багов.
